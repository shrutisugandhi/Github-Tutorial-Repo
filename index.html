Github Commands Usage Tutorial

Git is for tracking changes

  • Git create a repo which contain all your source code file
  • create a file & then commit to main/master branch means saved to local git repo .Always give proper commit msg of each commit.

    commit means this is final version of my code

  • publish repo means push repo to remote git origin
    publish/push code to remote repo for sharing with team using repo url
  • Push code to remote copy of your repo on Github(collection of all your repos) & so this is also called as open source contribution as all team member can perform content modification
  • markdown is one of feature of git which convert your text in Readme.md file into proper html file which is also understooble by git
  • Deploy this demo to netlify
  • after commit and push to github remote repo you will noticed deployment is also done at netlify once you refresh your netlify site.

    This above process of pushing code to repo & it is updated at your hosting site - called as continuous deployment

  • Create a branch named as -Dashboard branch ,

    Branch is similar to working on diff module/feature & once work completed (or as a backup of your work )save it on main branch at repo to avoid future loss of code at local machine

    Commit code & publish code on new branch

    Compare between main branch index.html & dashboard branch index.html, you will notice change in content on both branches ,so for this you have to merge your dashboard branch to main branch

    Also netlify uses main branch for deployment not any other branch ,if your code is not merged in main branch so it wont be shown neither at main branch not at netlify website

  • Open a PR(pull request) means : PR review, important when you're working in a team
    • create a pull request for your new branch
    • fill in description and headline
    • check out the file changes
    • get the PR link. you can send this for review to your friend to make sure that you're doing the right changes. That would essentially be PR review
